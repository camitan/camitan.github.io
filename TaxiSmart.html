TaxiSmart Application
Overview
User: IU students, Indiana residents
Location | Time: Indiana | Spring 2019
Language/Library: C#, Google Map API, SQL
Tools: Visual Studio (Windows Forms), database


This is a 3-person team project during the class Web Programming at IU. TaxiSmart application allows users to calculate taxi fare and show route by dropping pins/selecting locations and destinations on the map. The app will pull data from database based on usersâ€™ inputs and reserve an available cab from the database with invoice. This app is similar to Uber service, but we did not see any application for actual taxi service, and that's why we are very interested. Please read our full business idea report and detailed instruction here: Full Report , How to use this app, step by step.

Goal for the project
  • User can freely drop pins on the map and it will visualize preview route on map
  • Calculate taxi fare based on users' requirements
  • Check available cab in the database and update database after reservation.
  • User-centered interface design
Process


What was my role in the Team?
I was acting as a leader of this group project. As a start, I personally designed the whole interface with functions and features on paper as our prototype. After reviewing and discussing with group members, I coordinated the timeline of the project and assigned small assignments to each member every week. I also contributed a lot to map functions and database as well as problem-solving and debugging when we face difficulties.

What is the biggest challenge during this project?
I have to say we learned a lot during this project. From design and functions, we are glad they all works. However, nothing came easy. We have came through multiple challenges and limitations of the project. For example, We spent so much time discussing questions like 'what is the best way to choose locations on map' or 'how to delete pin and still save the previous pin, and how to delete route?'. Our project still has limitation like optimal route, if we have more time, we could expand the project to a real business application in the future.
