PLTW Style GuideBook Website
Overview
User: Project Lead The Way Inc. Production Team
Location | Time: Indianapolis, IN | Summer 2019
Language/Library: HTML, CSS, BootStrap, Animation
Tools: Adobe Illustrator, Whiteboard


Every company has its own rules about how to use their logo, color, font and media assets, in order to increase its brand visibility. PLTW Style GuideBook Website is created as a tool for internal employees when they have questions about styling and design. The main users of the site are designers, writers, or any media assets creaters in the company.

Goal for the project
  • User can quickly find the information and able to use them immediately
  • Clean design aligned with company style embeded
  • Help users understand efficiently
  • Make it responsive so user can access through phone.
Process


Design
First step, I designed on the white board and illustrator about the overall structure, navigation and homepage. Decided to put the most frequently used information on the home page: Logo, Font and color.
Building and Feedbacks
After making the first version of front page, I brought it to my colleagues and ask for their feedbacks. They really like my idea of using a turnable circle to copy the hexcode of color since it's so easy to use. Then I continued coding my next page. I also used Illustrator to create do's and don'ts logos, which can be more straightforward for user to see some common mistakes. Finaly, using Bootstrap to make it responsive.
Revise -> Change -> Test
The information on the page is overflowed! Manager also gave me more and more pieces of media assets information to add every day. So, I need to re-arrange the navigation, decided to make a mega dropdown menu, clearer, and pretier.
Outcome
What is the biggest challenge during this project?
I would say Design. Every day I wanted to change my design, tested different colors, just becasue I want to make it more delightful on visual. Dribbble and Awwwards are awesome resouces for my everyday inspiration. I was worried that I couldn't bring it to a 'visual-delight' website, but then I realized, why should I? Clean and clear is the key to a heavily informative website! Simple and easy to use are my goals so I should stick to it and be careful with colors. So later on, I put more efforts on increasing the smoothness of the using the website.

In addition, this is an individual project, so I need my courage and motivation to ask and test on real users. It can be a lot of works, but it's worth it.
